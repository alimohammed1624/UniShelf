# **UniShelf**

## **Features:**

1. Multi format resource sharing (like videos, video links, pdfs, codes, images, etc.,.) \[download to device or view the resource anytime\]

2. Code sharing support \[full code or code snippets, GitHub repo links, syntax highlighted code viewer, copy code functionality\]

3. University specific ecosystem (Only users with university email ids can register, content visible only within the same university, acts as a single academic hub for the university)  
   Filter using tags through departments/course names.

4. Privacy & Sharing controls (Public Resources, Private Resources, Selective Sharing \[share resources with specific users , groups, optional time limited access\]

5. Search & Filters ( search by: title, description, tags, code language, filters like resource type , subject, semester, popularity, date, etc.,.)

6. academic organization ( course-wise resources, sem-wise grouping, subject-specific categories, exam-oriented collections)

7. Option to upload anonymously but the moderator/admin can see which account uploaded what (to verify reporting)

8. Each asset or user can have tags

9. Users can have atomic blacklist/whitelist for who can access the resource  
   

**USP:**  
University specific.

Why would g-drive not be a good fit ?  
It is not centralized, removes the need for multiple links, being able to search for things that were not shared to you personally.

**Things not to include:**

- Automated content safety/moderation  
- User Engagement \[comments & discussion threads, likes and upvotes\]   
- group & community features \[groups and spaces like batch wise spaces, club & society spaces and project-specific groups, Seniors share notes & prep material, etc.,.\]   
- Code-centric features
